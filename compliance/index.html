Flagship

FacctList — Sanctions Intelligence

The Problem

  • Banks must act on sanctions as soon as press releases are published — but official lists lag. The gap is a compliance risk
  • Press releases come in mixed formats (tabular, free-form legal text) with entities, aliases, and dates embedded inconsistently
  • Previously, analysts manually processed press releases — days per update

High-Level Design

  • Monitor polls EU press releases and official journals for new sanctions publications
  • Extractor — format-aware parsing (tabular vs free-form legal text) into a unified LLM extraction step. Publications span tens to hundreds of pages. Alias extraction handles inconsistent name representations via prompt chaining
  • Structured records feed into FacctList for deduplication, reconciliation, and distribution to screening engines
Monitor → Extractor → FacctList ingestion pipeline

Results

96% — field-level extraction accuracy, evaluated over 5+ years of historical data
Days → minutes — press release to structured sanctions record
  • Integrated into FacctList's production pipeline, live with global clients
MVP

FacctGuard — Transaction Monitoring

The Problem

Banks flag suspicious transactions for AML at two levels: alert (is this transaction suspicious?) and SAR (should it be escalated to a regulatory filing?).

Approach

  • LSTM + XGBoost on 380M UK transactions — LSTM for sequential patterns (velocity, timing), XGBoost for tabular features (amounts, counterparty risk, aggregates)
  • Explainability: SHAP attributions → LLM-generated natural language summaries. Ops teams see “high-value transfer to new counterparty in high-risk jurisdiction” instead of feature vectors

Results

3x lift in recall — at fixed review capacity vs rule-based alert generation. The ops team reviews the same number of alerts but catches 3x more truly suspicious cases
Infrastructure

Private LLM Inference Stack

The Problem

Compliance systems handle sensitive financial data. Sending it to third-party LLM APIs is a regulatory non-starter for banking clients — but multiple systems depend on LLM capabilities.

Approach

  • Self-hosted Qwen-2.5-14B + vLLM — full inference stack on Facctum’s infrastructure
  • Qwen-2.5-14B: Apache 2.0 license, outperforms Mistral at scale, fine-tunable for regulatory documents
  • vLLM: continuous batching + PagedAttention for variable concurrent load

Results

Removed the data residency blocker — Facctum can serve regulated clients who would otherwise reject the product.
